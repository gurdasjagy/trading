PART 1: CRITICAL BUGS & ISSUES FOUND
1. Unsafe Static Mutable (UB in Rust)
In main.rs, the strategy evaluator loop uses static mut SNAPSHOT_COUNTER with unsafe block. This is undefined behavior in Rust since the strategy thread could theoretically be accessed from multiple contexts. Replace with AtomicU64.
2. Lifetime Annotation Error
strategy_evaluator_loop has &'regime_shm::SharedMemRegimeReader which is a malformed lifetime annotation. This should be &regime_shm::SharedMemRegimeReader (a regular reference) or use a proper named lifetime like &'a regime_shm::SharedMemRegimeReader.
3. Unresolved config.symbols in Execution Router
In execution_router_loop, the code references config.symbols and config::symbols but config is not passed as a parameter to this function. The set_margin_mode loop and funding rate fetch both reference symbols that don't exist in scope. This will cause compilation failures.
4. exec_analytics Double Lock Deadlock
In the strategy evaluator, exec_analytics is locked via exec_analytics.lock() at initialization and stored in a local variable, but then later exec_analytics.lock() is called again inside the loop body. Since parking_lot::Mutex is not reentrant, this causes a deadlock on the first iteration.
5. MicrostructureMetrics String Lifetime Issues
wyckoff_phase and ichimoku_cloud_position are typed as &'static str but are populated with .to_string() results that are then referenced. The to_string() calls in main.rs create owned Strings that can't be coerced to &'static str without leaking memory.
6. Duplicate exec_analytics Allocation
In main(), both exec_analytics_arc (Arc) and exec_analytics (&'static leaked) are created independently with separate ExecutionAnalytics::new(1000) calls. The strategy thread and execution thread operate on different instances, so execution analytics data is never shared.
7. recent_highs/recent_lows Reset Every Tick
In the strategy evaluator, recent_highs and recent_lows are declared inside the loop body, so they're reset to empty vectors on every snapshot. CVD divergence detection (detect_bearish_divergence, detect_bullish_divergence) will never have enough data points to detect anything.
8. TWAP Executor Never Receives Prices
twap_exec.tick() is called with an empty HashMap::new() for current_prices. TWAP slices will never have valid prices and will either fail or submit at price 0.
9. Position Entry PnL Tracking Bug
In execution_router_loop, position_entries.insert() is called for every fill, but position_entries.remove() is also called in the same block. For new position entries, the remove returns None (correct), but for position closes, the entry was already overwritten by the new fill's insert. PnL calculation is unreliable.
10. Orderbook Builder Spin Loop
orderbook_builder_loop uses std::hint::spin_loop() when the ring is empty, which burns 100% CPU on the pinned core. Should use a brief thread::yield_now() or thread::sleep(Duration::from_micros(1)) for non-HFT workloads.
