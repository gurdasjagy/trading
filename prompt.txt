unified_step = max(spot_spec.step_size, futures_spec.step_size_in_base_asset)
final_qty = floor(raw_qty / unified_step) * unified_step

### PHASE 4: EXECUTION ENGINE

#### New File: `rust_engine/src/multi_exchange/spot_futures_executor.rs`

**Maker-Taker Sniper Execution Pattern:**

```
ENTRY:
1. Identify the less liquid leg (usually Spot has thinner books)
2. Place Post-Only (Maker) limit order on Spot at Best Bid + 1 tick
3. Listen for fill via private execution stream (WS user data on Binance/Bybit, REST poll on Gate.io)
4. The MILLISECOND the Spot order fills (partial or full):
   -> Fire Market (IOC) order on Futures for the EXACT filled quantity
5. If Spot doesn't fill within timeout (30s), cancel it (zero exposure, zero risk)
6. If Futures hedge fails after Spot filled:
   -> Immediately market-sell the Spot to close exposure
   -> Log CRITICAL alert
   -> Record the loss from spread

EXIT (reverse):
1. Place Limit order to Buy-to-Close the Futures Short
2. The millisecond it fills, Market-Sell the Spot holdings
3. If Spot sell fails after Futures close:
   -> You still hold the crypto, which has value
   -> Retry the Spot sell with wider slippage
   -> This is less dangerous than the entry case
```

**Spot order differences from Futures (the coding agent MUST handle these):**
- No `leverage` parameter (Spot is always 1x)
- No `reduce_only` (Spot doesn't have this concept)
- Quantity is in BASE asset (e.g., 0.001 BTC), not USDT and not integer contracts
- You receive the actual crypto asset in your Spot wallet
- To sell Spot, you must have the asset in your Spot wallet (not USDT)
- Gate.io Spot uses `amount` field (base qty), NOT `size` (integer contracts)
- Binance market buys should use `quoteOrderQty` (spend X USDT) for safer sizing

### PHASE 6: EXIT LOGIC

#### In: `rust_engine/src/multi_exchange/spot_futures_engine.rs`

**Exit Triggers (checked every 30 seconds):**

1. **Negative Funding**: If funding rate turns negative for 2 consecutive periods, shorts start PAYING longs. Exit immediately.

2. **Basis Reversion**: If entered when Futures were premium to Spot, and now Futures are at or below Spot, close to capture the basis as profit on top of funding.

3. **Accumulated Profit Target**: When `accumulated_funding > capital_deployed * take_profit_pct` (configurable, default 0.5%).

4. **Margin Danger**: If Futures margin ratio drops below 15% even after rebalancing attempts.

5. **Time Stop**: Maximum hold time (configurable via `SPOT_FUTURES_MAX_HOLD_HOURS`, default 72).

6. **Kill Switch**: If `kill_switch.rs` is triggered, immediately unwind all positions.

**Unwind Execution:**
```
1. Place Limit order to Buy-to-Close the Futures Short
2. The millisecond it fills, Market-Sell the Spot holdings
3. Calculate final PnL:
   final_pnl = accumulated_funding
              + (futures_entry - futures_exit) * qty  // Basis capture
              + (spot_exit - spot_entry) * qty        // Should be ~0
              - total_fees
4. Log: symbol, hold_time, funding_collected, basis_pnl, fees, net_pnl
5. Persist closed position to SQLite
6. Release position slot (allow new positions)
```
