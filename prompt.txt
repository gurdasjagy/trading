### BUG 1: Gate.io WS Authentication - "Not Login" Error on Order Placement

**File:** `rust_engine/src/gateio_gateway.rs`

**Problem:** Gate.io WS v4 requires an explicit `login` channel authentication BEFORE any `futures.order_place` API calls. When `submit_order` sends a `futures.order_place` message, Gate.io rejects it with `WS_REJECT: Not login`.

```rust

let login_msg = format!(
    r#"{{"time":{},"channel":"futures.login","event":"api","payload":{{"api_key":"{}","signature":"{}","timestamp":"{}"}}}}"#,
    login_time, api_key, login_sign, login_time
);
info!("[gateio-ws] Sending explicit login authentication");
if let Err(e) = ws_write.send(Message::Text(login_msg)).await {
    error!("[gateio-ws] Login send failed: {}", e);
    continue;
}

// Wait for login success response
let login_deadline = Instant::now() + Duration::from_secs(5);
let mut login_success = false;
while Instant::now() < login_deadline {
    tokio::select! {
        msg = ws_read.next() => {
            match msg {
                Some(Ok(Message::Text(txt))) => {
                    if txt.contains("futures.login") && 
                       (txt.contains("\"status\":\"success\"") || txt.contains("\"error\":null")) {
                        info!("[gateio-ws] Login authentication successful");
                        login_success = true;
                        break;
                    } else if txt.contains("futures.login") && txt.contains("error") {
                        error!("[gateio-ws] Login authentication failed: {}", txt);
                        break;
                    }
                }
                _ => {}
            }
        }
        _ = tokio::time::sleep(Duration::from_millis(100)) => {}
    }
}

if !login_success {
    warn!("[gateio-ws] Login failed, reconnecting...");
    tokio::time::sleep(Duration::from_millis(backoff_ms)).await;
    backoff_ms = (backoff_ms * 2).min(RECONNECT_MAX_MS);
    continue;
}

// Step 2: THEN subscribe to futures.orders (existing code)
```

### BUG 3: USDT-to-Contracts Conversion Broken for Multi-Exchange

**Files:** `rust_engine/src/binance_gateway.rs`, `rust_engine/src/bybit_gateway.rs`

**Problem:** When the strategy engine emits `size=1` (meaning 1 contract on Gate.io where 1 contract = 0.0001 BTC), Binance interprets `qty=1` as 1 whole BTC ($70,000+), causing "insufficient balance". The size normalization between exchanges is fundamentally broken.

**Fix:** Implement a universal size normalization layer:

```rust
// In a new file: rust_engine/src/size_normalizer.rs

pub struct SizeNormalizer;

impl SizeNormalizer {
    /// Convert a USDT notional value to exchange-specific contract quantity.
    /// 
    /// Gate.io: contracts = usdt_notional / (quanto_multiplier * price)
    /// Binance: quantity = usdt_notional / price (in base currency units)
    /// Bybit:   quantity = usdt_notional / price (in base currency units)
    pub fn usdt_to_exchange_qty(
        exchange: ExchangeId,
        usdt_notional: f64,
        price: f64,
        quanto_multiplier: f64,
        step_size: f64,
        min_qty: f64,
    ) -> Result<f64, String> {
        if price <= 0.0 { return Err("Invalid price".into()); }
        
        let raw_qty = match exchange {
            ExchangeId::GateIo => {
                if quanto_multiplier > 0.0 {
                    usdt_notional / (quanto_multiplier * price)
                } else {
                    usdt_notional / price
                }
            }
            ExchangeId::Binance | ExchangeId::Bybit => {
                usdt_notional / price
            }
        };
        
        // Snap to step size
        let snapped = if step_size > 0.0 {
            (raw_qty / step_size).floor() * step_size
        } else {
            raw_qty
        };
        
        if snapped < min_qty {
            return Err(format!("Qty {} below minimum {}", snapped, min_qty));
        }
        
        Ok(snapped)
    }
}
```

### BUG 5: Funding Arb Legging Risk - Naked Directional Exposure

**Files:** `rust_engine/src/multi_exchange/funding_arb_executor.rs`, `rust_engine/src/multi_exchange/funding_arb_engine.rs`

**Problem:** The bot fires two simultaneous market orders for funding arb (short on high-rate exchange, long on low-rate exchange). If one leg fails, the bot has naked directional exposure. There is no atomic execution or rollback mechanism.

**Fix:** Implement a Maker-Taker leg execution pattern:

```rust
// In funding_arb_executor.rs:

pub struct DualLegExecutor {
    // ... existing fields ...
}

impl DualLegExecutor {
    /// Execute a funding arb trade with proper leg sequencing.
    /// 
    /// 1. Place MAKER (post-only) order on the less liquid exchange first
    /// 2. Wait for fill confirmation (poll every 100ms, timeout 30s)
    /// 3. Only when leg 1 fills, immediately hedge with IOC taker on liquid exchange
    /// 4. If leg 1 doesn't fill within timeout, cancel it (no exposure)
    /// 5. If leg 2 fails after leg 1 filled, immediately close leg 1 at market
    pub async fn execute_with_leg_protection(
        &self,
        maker_exchange: ExchangeId,
        taker_exchange: ExchangeId,
        symbol: &str,
        maker_side: OrderSide,
        taker_side: OrderSide,
        size_usdt: f64,
        maker_price: f64,
    ) -> Result<DualLegResult, ExchangeError> {
        // Step 1: Place maker order
        let maker_gw = self.get_gateway(maker_exchange)?;
        let maker_qty = self.normalize_qty(maker_exchange, symbol, size_usdt, maker_price).await?;
        
        let maker_intent = OrderIntent {
            symbol: symbol.to_string(),
            side: maker_side.clone(),
            size: maker_qty,
            order_type: OrderType::PostOnly,
            price: Some(maker_price),
            reduce_only: false,
            leverage: Some(3),
            time_in_force: "poc".to_string(),
            ..Default::default()
        };
        
        let maker_result = maker_gw.submit_order(maker_intent).await?;
        
        // Step 2: Poll for fill (max 30 seconds)
        let mut filled = false;
        let mut fill_price = 0.0;
        let mut fill_size = 0i64;
        let deadline = Instant::now() + Duration::from_secs(30);
        
        while Instant::now() < deadline {
            tokio::time::sleep(Duration::from_millis(100)).await;
            if let Ok(Some(status)) = maker_gw.get_order_status(&maker_result.order_id, symbol).await {
                if status.filled_size > 0 {
                    filled = true;
                    fill_price = status.avg_fill_price;
                    fill_size = status.filled_size;
                    break;
                }
                if status.status == "cancelled" || status.status == "expired" {
                    break;
                }
            }
        }
        
        if !filled {
            // Cancel unfilled maker order - no exposure
            let _ = maker_gw.cancel_order(&maker_result.order_id, symbol).await;
            return Err(ExchangeError::Unknown {
                code: "MAKER_NOT_FILLED".into(),
                message: "Maker leg did not fill within timeout".into(),
            });
        }
        
        // Step 3: Hedge with IOC taker on liquid exchange
        let taker_gw = self.get_gateway(taker_exchange)?;
        let taker_qty = self.normalize_qty(taker_exchange, symbol, fill_size as f64 * fill_price, fill_price).await?;
        
        let taker_intent = OrderIntent {
            symbol: symbol.to_string(),
            side: taker_side.clone(),
            size: taker_qty,
            order_type: OrderType::Market,
            price: None,
            reduce_only: false,
            leverage: Some(3),
            time_in_force: "ioc".to_string(),
            ..Default::default()
        };
        
        match taker_gw.submit_order(taker_intent).await {
            Ok(taker_result) => {
                Ok(DualLegResult {
                    maker_leg: LegStatus::Filled { 
                        exchange: maker_exchange, 
                        price: fill_price, 
                        size: fill_size 
                    },
                    taker_leg: LegStatus::Filled { 
                        exchange: taker_exchange, 
                        price: taker_result.avg_fill_price, 
                        size: taker_result.filled_size 
                    },
                })
            }
            Err(e) => {
                // CRITICAL: Taker failed - close maker leg immediately
                error!("[funding-arb] TAKER LEG FAILED - closing maker leg at market: {}", e);
                let close_side = if maker_side == OrderSide::Buy { OrderSide::Sell } else { OrderSide::Buy };
                let close_intent = OrderIntent {
                    symbol: symbol.to_string(),
                    side: close_side,
                    size: fill_size,
                    order_type: OrderType::Market,
                    reduce_only: true,
                    ..Default::default()
                };
                let _ = maker_gw.submit_order(close_intent).await;
                Err(e)
            }
        }
    }
}
```

   - Bybit: 8-hour intervals (00:00, 08:00, 16:00 UTC)  
   - Gate.io: 8-hour intervals (00:00, 08:00, 16:00 UTC)
2. Add a `FundingTimingEngine` that:
   - Calculates time until next funding snapshot for each exchange
   - Only opens positions within a configurable window before snapshot (default: 60 seconds)
   - Automatically closes positions within 30 seconds after funding is credited
   - Tracks cumulative funding received per position

```rust
// New file: rust_engine/src/funding_timing.rs

use std::time::{SystemTime, UNIX_EPOCH, Duration};

pub struct FundingTimingEngine {
    /// Seconds before funding snapshot to enter position
    entry_window_secs: u64,
    /// Seconds after funding snapshot to exit position
    exit_window_secs: u64,
    /// Funding intervals per exchange (in hours from midnight UTC)
    funding_hours: Vec<u64>, // [0, 8, 16] for most exchanges
}

impl FundingTimingEngine {
    pub fn new() -> Self {
        Self {
            entry_window_secs: 60,
            exit_window_secs: 30,
            funding_hours: vec![0, 8, 16],
        }
    }
    
    /// Returns seconds until next funding snapshot
    pub fn seconds_until_next_funding(&self) -> u64 {
        let now = SystemTime::now()
            .duration_since(UNIX_EPOCH)
            .unwrap_or_default()
            .as_secs();
        let current_hour = (now / 3600) % 24;
        let current_minute_secs = now % 3600;
        
        for &funding_hour in &self.funding_hours {
            if funding_hour > current_hour || 
               (funding_hour == current_hour && current_minute_secs < 3600) {
                return (funding_hour - current_hour) * 3600 - current_minute_secs;
            }
        }
        // Next day's first funding
        (24 - current_hour + self.funding_hours[0]) * 3600 - current_minute_secs
    }
    
    /// Returns true if we're within the entry window before funding
    pub fn is_entry_window(&self) -> bool {
        self.seconds_until_next_funding() <= self.entry_window_secs
    }
    
    /// Returns true if we should exit (funding just occurred)
    pub fn is_exit_window(&self) -> bool {
        let secs = self.seconds_until_next_funding();
        // Just after funding = close to next 8h mark
        let time_since_last = 8 * 3600 - secs;
        time_since_last <= self.exit_window_secs
    }
}
```

### FEATURE 5: Position Delta Neutrality Monitor

**File:** New file `rust_engine/src/multi_exchange/delta_monitor.rs`

**Problem:** In funding arb, the bot should maintain delta-neutral exposure (net position across all exchanges = 0). Currently there's no monitoring or auto-correction for delta drift.

```rust
pub struct DeltaNeutralityMonitor {
    max_delta_usdt: f64,  // Maximum allowed net delta in USDT
    check_interval_secs: u64,
}

impl DeltaNeutralityMonitor {
    /// Calculate net delta across all exchanges for a symbol
    pub async fn calculate_net_delta(
        &self,
        symbol: &str,
        gateways: &HashMap<ExchangeId, Arc<dyn ExecutionGateway>>,
    ) -> f64 {
        let mut net_delta = 0.0;
        for (_, gw) in gateways {
            if let Ok(Some(pos)) = gw.get_position(symbol).await {
                let delta_usdt = pos.size as f64 * pos.entry_price;
                net_delta += delta_usdt;
            }
        }
        net_delta
    }
    
    /// Check if delta is within acceptable range
    pub fn is_delta_neutral(&self, net_delta: f64) -> bool {
        net_delta.abs() < self.max_delta_usdt
    }
    
    /// Generate hedge order to restore delta neutrality
    pub fn generate_hedge_order(
        &self,
        net_delta: f64,
        symbol: &str,
        price: f64,
    ) -> Option<OrderIntent> {
        if self.is_delta_neutral(net_delta) {
            return None;
        }
        
        let side = if net_delta > 0.0 { OrderSide::Sell } else { OrderSide::Buy };
        let size = (net_delta.abs() / price).ceil() as i64;
        
        Some(OrderIntent {
            symbol: symbol.to_string(),
            side,
            size,
            order_type: OrderType::Market,
            price: None,
            reduce_only: false,
            leverage: Some(3),
            time_in_force: "ioc".to_string(),
            slippage_cap_pct: Some(0.002),
            placement: PlacementType::AtBest,
            stop_loss: None,
            take_profit: None,
            confidence: 1.0,
            signal_tag: "delta_hedge".to_string(),
```
