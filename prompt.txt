# Requirements: 5 Trading Bot Features
## Feature 1: Position Reconciliation on Startup
- On startup, query Gate.io REST API for all open positions via `get_positions()`
- For each open position found on exchange, register it in the `ExitEvaluator` and `PositionLifecycleManager`
- Apply auto-protection (configurable SL/TP) to any position that doesn't have exchange-side conditional orders
- Must happen AFTER gateway auth is verified but BEFORE the main execution loop starts
- Implemented in `execution_router_loop()` in main.rs, right after the initial balance check
## Feature 2: Configurable Auto-Protection SL/TP
- Add `AutoProtectionConfig` struct to config.rs with fields:
  - `default_sl_pct: f64` (default 2.0 = 2%)
  - `default_tp_pct: f64` (default 3.0 = 3%)
  - `use_atr_based: bool` (default false)
  - `atr_sl_multiplier: f64` (default 2.0, used when use_atr_based=true)
  - `atr_tp_multiplier: f64` (default 3.0, used when use_atr_based=true)
  - `scan_interval: u64` (default 50, how often to scan for unprotected positions)
- Add to `EngineConfig` struct
- Pass to strategy_evaluator_loop and use in auto-detection logic
- Replace hardcoded 2%/3% values with config values
## Feature 3: Exchange-Side Conditional Orders for Auto-Protected Positions
- When auto-detection applies SL/TP to a position, also submit exchange-side conditional orders
- Need a channel from strategy thread to execution thread to request SL/TP submission
- Use existing `crossbeam_channel` for this communication
- Create `SlTpUpdateRequest` struct with symbol, side, size, sl_price, tp_price
- In execution_router_loop, check channel and submit via `gateway.submit_conditional_sl/tp()`
## Feature 4: Persistent State (SQLite)
- Add `rusqlite` dependency to Cargo.toml
- Create `state_store.rs` module with `StateStore` struct
- Tables: `positions` (symbol, side, entry_price, size, sl, tp, opened_at), `orders` (order_id, symbol, status, price, size, created_at)
- On startup: load persisted positions and reconcile with exchange state
- On position open/close/update: write to SQLite
- DB path configurable via config (default: `./trading_state.db`)
## Feature 5: Exchange-Side SL/TP Updates for Trailing Stops
- Add `cancel_conditional_orders()` and `update_conditional_sl()` methods to ExecutionGateway trait and GateIoGateway
- Gate.io requires: cancel old conditional order, then submit new one at updated price
- Wire into trailing stop update path in strategy_evaluator_loop (lines 1634-1654 where TODO comments exist)
- Use the same crossbeam channel from Feature 3 for async SL/TP updates
- Track exchange-side conditional order IDs so we can cancel them when updating
